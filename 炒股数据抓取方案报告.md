# 炒股数据抓取方案报告

**软件**：野狼3000（鼎信汇金，桌面客户端，主要用于股指期货分析）  
**需求**：每隔固定时间自动抓取页面数据，存入数据库  
**日期**：2026-05-27

---

## 方案对比

| 方案 | 原理 | 难度 | 稳定性 | 费用 |
|---|---|---|---|---|
| 方案一：Fiddler 抓包 | 拦截软件的 HTTP 网络请求，找到数据接口直接调用 | 低 | 高 | 免费 |
| 方案二：Windows UI Automation | 用 pywinauto 读取界面控件的值，不需要截图 | 中 | 高 | 免费 |
| 方案三：截图 + OCR | 定时截图，用 OCR 识别数字 | 中 | 中（约95~99%准确率） | 免费 |

**推荐优先级**：方案一 → 方案二 → 方案三

---

## 方案一：Fiddler Classic 抓包（当前进行中）

### 工具说明

| 工具 | 费用 |
|---|---|
| Fiddler Classic | **免费** |
| Wireshark | **免费** |
| Charles | 付费（$50），有30分钟限制试用 |

> Windows 推荐使用 **Fiddler Classic**，完全免费。

### 操作步骤

**第一步：下载安装**
1. 搜索 `Fiddler Classic download` 或访问 `telerik.com`
2. 下载时填邮箱（随便填），默认选项安装

**第二步：开启 HTTPS 解密**
1. 顶部菜单 `Tools` → `Options...`
2. 点 `HTTPS` 标签页
3. 勾选 `Capture HTTPS CONNECTs`
4. 勾选 `Decrypt HTTPS traffic`
5. 弹出安装证书提示 → 全部点 `Yes` 确认
6. 点 `OK` 关闭

**第三步：抓包**
1. 保持 Fiddler 开着
2. 打开野狼3000，让行情正常刷新
3. 观察 Fiddler 左侧请求列表，寻找包含以下关键词的 URL：
   - `quote` / `market` / `tick` / `kline` / `realtime`
4. 点击请求条目，右侧下半部分点 `JSON` 标签
5. 如果能看到股票代码和价格数字，说明抓包成功

**第四步：记录以下信息（供后续写代码用）**
- 请求的完整 URL
- 请求方式（GET 还是 POST）
- 请求头里有没有 token / cookie / 签名参数
- 返回数据的格式（JSON / XML / 其他）
- 请求频率（软件多久请求一次）

### 抓包失败的情况

如果 Fiddler 左侧列表为空，或返回内容全是乱码，说明：
- 软件使用了非 HTTP 协议（如 TCP 自定义协议）
- 数据在本地计算，不从网络获取

此时改用**方案二**。

## 方案二：Windows UI Automation

**工具**：Python + `pywinauto`

**验证是否可行**：
1. 下载微软 Inspect.exe（Windows SDK 自带，或单独搜索下载）
2. 打开 Inspect.exe，鼠标悬停到野狼3000 界面上的数字
3. 查看右侧面板中是否显示 `Name` 或 `Value` 属性及其数值
4. 如果能读到，说明此方案可行

---

## 方案三：截图 + OCR（兜底）

**工具**：
- `mss`：Python 截图库
- `ddddocr` 或 `PaddleOCR`：国产 OCR，对金融数字识别效果好

**注意**：需要软件窗口位置固定，不能移动。

---

## 数据库选型

| 场景 | 推荐 |
|---|---|
| 简单存储，个人使用 | SQLite（无需安装服务） |
| 数据量大、需要查询分析 | PostgreSQL |
| 专门的时序数据 | InfluxDB / TimescaleDB |
